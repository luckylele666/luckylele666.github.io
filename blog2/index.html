写 Blog 的大坑及教训

本文最后更新于：September 2, 2020 am

手拆了石膏后，终于可以双手打字，这几天写了些 blog，也算是对这个荒废的夏天的一些挽救吧。前天才发现 Hexo 的渲染器做的不好，英文的 「'」在实际中显示为 ‘，Tim 教我装了另一个渲染器——hexo-renderer-markdown-it 来替换 hexo-renderer-markded，设置 typographer: false，但是设置后也没有改变。

升级 Hexo

我决定升级 Hexo 及配套环境，直接在官网找 Git（还有 portable 版） 和 Node.js 进行覆盖安装（没有上网环境的安装「淘宝」镜像包），一路点默认的选项安装以免出错。

我升级了 Hexo，升级倒不麻烦，只需要在 官网 找到命令就好了，官网的文档并不好，我按照网上教程来，放在这里，npm = $。

  1. 全局升级 hexo-cli

    npm install hexo-cli -g

  2. 检查插件升级，查看安装的插件

    npm install -g npm-check
npm-check

  3. 升级插件

    npm install -g npm-upgrade
npm-upgrade

  4. 更新全局包

    npm update -g

  5. 更新生产环境依赖包

    npm update --save

  6. 查看 Hexo 版本

    $ hexo v

后面就有一长串安装成功的版本信息，

在升级插件的过程中会有提示 Yes or No，只需要连续回车键。安装中让人难受的是没有进度条，等待的过程比较焦急，光标若是在闪烁且没有指令出来就是正在加载……升级到最新的 5.1.1（GitHub 上的最新版），渲染问题也没有解决，后来我找到使用 \' 这个反义语法来解决。

升级 Melody 主题

之前（并不是这个）的 Melody 也有升级了，进入官方文档，（Hexo 版本 >= 5.0）只需要键入

npm install hexo-theme-melody

即可 npm「一键」升级。半夜升级完，我也没再测试，第二天要发三星 T7 移动硬盘的那篇文章才发现遇到大问题了，Git 提示：err: Error: Spawn failed。什么，产卵失败？！当时不够耐心，没有搜索关键词「Spawn failed」以至于我好久没找到解决方法，又重新装了个安装版 Git，未果。暂时搁置，等到晚上再次尝试，重装 Hexo！

首先创建一个文件夹放在自己喜欢的位置，我犯了个错误，直接在桌面开始新建了，后面移动更改了路径，为了避免未知的错误，务必从一而终的安装在同一个目录下。

先检查下自己的版本号：

$ git version
$ node -v
$ npm -v

准备好开始安装 Hexo，输入：

npm install -g hexo-cli

安装时间较长耐心等待，初始化博客，输入：

hexo init blog

之后会出现 _config.yml（俗称站点配置），package.json，source（资源文件夹）等等，除了 _posts 文件，其他下划线 _ 开头的不会被编译打包到 public 文件夹。再输入：

hexo new test_my_site

hexo g

hexo s

s 是 server 的缩写，打开浏览器输入 http://localhost:4000 就能成功打开第一个默认主题 landscape。

最核心的三条命令：

hexo g = hexo generate 生成
hexo s = hexo server 启动服务预览
hexo d = hexo deploy 部署

在每个步骤间穿插 hexo clean （清除缓存）确保不会出错。

还有可能用到的命令：

hexo server -p 1xxx （更改端口）
hexo server -i 192.168.x.x （自定义 IP）

核心命令的作用是新建博客文章、生成网页、本地预览。

连接 Github Error

没有账号的话可以搜下相关教程。一个同名仓库只能托管一个静态站点（相当于一个 Blog），按理说重装 Hexo 不需要重新配置 SSH Key，但是我还是重置了一下，虽然给的 Key 是一样的。

以前有的话直接输入：

cat ~/.ssh/id_rsa.pub

查看本机密钥。没有的话先将 Git 和 GitHub 账号绑定：

git config --global user.name "GitHub 用户名"
git config --global user.email "GitHub 注册邮箱"

再生成 ssh 密钥：

ssh-keygen -t rsa -C "GitHub 注册邮箱"

然后三个回车，不需要设置密码，在 .ssh 文件夹里的 id_rsa.pub 文件复制里面全部内容，打开 GitHub_Settings_keys 页面，点 New SSH key，Title 随便写，然后在 Git Bash 检测SSH key 设置是否成功，输入 ssh -T git@github.com，原理是公钥放在 GitHub，私钥放在电脑里。GitHub 每次推送都要验证是不是你自己，为了方便，采用了 ssh，所以需要私钥跟 GitHub的公钥配对，然而我就遇到了无法配对的情况，这是第二个大坑！

「ssh -T git@github.com ssh: connect to host github. com port 22: Connection timed out」显示超时，GitHub 被污染严重，国内用户常常连不上……

我后来找到的解决方法：

  1. 打开 _config.yml 配置文件

  2. 拉到最下，改成：

deploy:

type: 'git'

repo: git@github.com:yourname/yourname.github.io.git

branch: master

其实这个问题也很简单，但是我没有仔细 Google，或者说以为自己的问题不是这样解决的，浪费了大量的时间，以后遇到问题应该什么方法都试一下的，反正已经出错了也不怕再搞坏。

然后绑定自己的顶级域名 具体操作 不赘述，不想花钱买也可以用 GitHub 的。

Fatal

想着试下广受好评的 NexT 主题，直接 npm install hexo-theme-next，我打开 theme 没有看到文件夹以为安装失败，随后才知道，Hexo 5.0 之后是更科学的安装方式。主题文件夹位于 node_modules，所以安装前还要看下主题的说明文档（看资料的重要性）在接下来的步骤我居然又遇到了「Spawn failed」的问题！再次搜索找到使用 git reset --hard commitId 的方法，这个 commitId 让我搞不懂，后来又在高人指点下才知道这就是一串 GitHub 上的 commit 的 sha 码，但是使用无效，在写这篇 blog 时候又看到有人说 git push -f 有效（未验证），就此陷入死循环，我又暂时搁置，看看教学视频缓解下急躁的心情。后来在 Hexo 博客群友的再次提醒下检查配置文件，发现这段：

deploy:

type: 'git'

repo: git@github.com:yourname/yourname.github.io.git

branch: master

里面居然有个中文？！去掉后万事大吉，天哪，这可真是太为难我了。

新主题

熬到很晚，睡了一觉，起床已经是我最爱的九月了！之前的一波三折已经磨损我的耐心，我决定再换个新主题来调节下心情，看了一圈，喜欢卡片式设计选了现在这个主题 Fluid，虽然没给我有液体的感觉，但是够简洁，之前就注意过，最近才更新，就你了。虽然没有 npm 安装，但是压缩包解压到 themes 不会出错，在安装过程中又报错，我心里咯噔一下，但是经过之前的波折，平稳了许多，根据报错提示，是 Melody 和 NexT 主题插件加载失败错误，搜索后：

查看插件

npm list

有一大串，找到 Melody 和 NexT 主题插件的名称，复制，卸载

npm uninstall hexo- pluginname

还需要删除相关配置和文件：

  • 配置文件中 Plugins 模块对应的插件设置
  • node_modules/ 目录下对应的插件文件

之后不再报错，然而我又遇到了 Big Trouble！进入 blog 点击文章出现下载文件而不是进入文章的情况。这下搜索没辙了，只好请教高手们，问了一下午，在 Fluid 作者之一的 张凯强 处得到了最准确的解决方法。原来我每篇文章的 permalink 的结尾都是不加「/」的，就差这一个斜杠，问题就解决了！你说奇妙不奇妙。

至此我终于是把 blog 新主题装好，部署，真的累的都没有创作的欲望了，作者还告知我一些不曾注意的细节，在配置文件里如有空格或是英文的输入都需要加上引号，若主题配置未生效还要检查站点配置，因为覆盖安装了，主题的优先级是大于站点配置的，但是有些站点配置却并未修改。还有标题旁边那个回形针🖇一样的东西是叫「锚点」，不喜欢也可以关闭。当我解决问题后，Tim 也想起来他也遇到过升级 Hexo 5.0 打开博客就是下载，不过都需要经历这个过程吧，考验我的耐心和阅读能力。

总结

做什么事都不能心急，一目十行，特别是关于前端代码这种我不擅长的事，需要：

  1. 好的提问和搜索能力
  2. 英语阅读能力
  3. 坚韧性💪

感谢在我焦急的时候给予帮助的人们🙏

P.S. 已经连续熬夜一个月，该早点洗洗睡了，我又不是程序员……
